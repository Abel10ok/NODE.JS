----
HTTP
ESTE PROTOCOLO DEFINDE EL FORMATO DE LOS MENSAJES.
UN PROTOCOLO ES UN CONJUNTO DE REGLAS QUE PERMITEN TRANSMITIR INFO ENTRE DISPOSITIVOS DE UNA RED.

-------
HEADERS
PROVEEN INFO ADICIONAL SOBRE LA SOLICITUD.

----
BODY
CONTIENE LA INFO QUE DEBE SER ENVIADA AL SERVIDOR PARA PROCESAR LA SOLICITUD, NO SE INCLUYE EN TODAS LAS SOLICITUDES, SOLO EN AQUELLAS QUE REQUIEREN ENVIAR INFORMACION, EJEMPLO: POST, PUT.

------------
METODOS HTTP
VERBO O SUSTANTIVO QUE INDICA LA INTENCION DE LA SOLICITUD.
    -GET:
    USADO PARA SOLICITAR UN RECURSO ESPECIFICO, EJEMPLO: ARCHIVO HTML, CSS O UNA IMAGEN.

    -POST:
    PARA CREAR UN RECURSO ESPECIFICO, EJEMPLO: AGREGAR UN USUARIO NUEVO A UNA BASE DE DATOS.

    -PUT:
    MODIFICAR UN RECURSO ESPECIFICO, EJEMPLO: HACER CAMBIOS EN LA BASE DE DATOS.

    -DELETE
    ELIMINAR UN RECURSO ESPECIFICO, EJEMPLO: ELIMINAR USUARIO DE BASE DE DATOS.
    -OTROS

RESPUESTAS HTTP
    HEADERS SON OPCIONALES Y PROVEEN INFORMACION ADICIONAL SOBRE LA RESPUESTA.

    BODY CONTIENE LA INFORMACION QUE DEBE SER ENVIADA DESDE EL SERVIDOR HACIA EL CLIENTE.

----------------------
CODIGOS DE ESTADO HTTP
NUMEROS QUE INDICA SI SE A COMPLETADO EXITOSAMENTE O NO LA SOLICITUD HTTP .
-INFORMATIVAS(100-199)
-SATIFACTORIAS(200-299)
-REDIRECCIONES(300-399)
-ERRORES DE CLIENTES (400-499)
-ERRORES DE LOS SERVIDORES(500-599)


CON NODE.JS Y EXPRESS PODEMOS ESPECIFICAR EL CODIGO DE ESTADO DE LA RESPUESTA HTTP EN NUESTRO SERVIDOR.

CODIGOS ESTADOS MAS COMUNES.
    -200 OK (LA RESPUESTA FUE EXITOSA).
    -400 BAD REQUEST (EL SERVIDOR NO PUDO INTERPRETAR LA SOLICITUD).
    -404 NOT FOUND (EL SERVIDOR NO PUDO ENCONTRAR EL RECURSO SOLICITADO).
    -500 INTERNAL SERVER ERROR (EL SERVIDOR ENCONTRO UNA SITUACION QUE NO SABE COMO MANEJAR).

------
PUERTO
UBICACION VIRTUAL DEL SISTEMA OPERATIVO EN LA CUAL SE PUEDE ACCEDER A UNA APP O UN PROCESO ESPECIFICO QUE SE ESTE EJECUTANDO EN ESE PUERTO.
